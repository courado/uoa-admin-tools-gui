Dashboard

The OMTD registry will be a resource based registry , i.e., it will try to agnostically manage resources whose schema is not necessarily known a priori. An OMTD resource is defined as any object that must be presented (viewed, searched, linked) as an autonomous entity in the OMTD space and users.

Brief description of functionality requirements

  • Extensibility on the resource types: ability to add/remove resource types on the fly, without changes on the underlying code and database schema.
  • Versioning
  • Registration and validation mechanisms for each resource type
  • Referential integrity among related resources
  • Different exchange formats: ability to store resources formatted in both JSON and XML.
  • REST/SOAP APIs for different types of searches (also based on relationships?)
  • Subscription/Notification (S/N): notify interested users and services for changes in the stored resources.
  • Scalability and performance
